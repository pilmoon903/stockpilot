# ADR-001: 오케스트레이션 도구로 n8n 선택

**날짜:** 2026-04-16
**상태:** Accepted
**관련:** PRD v1.5 섹션 5 (기술 스택)

---

## 컨텍스트

StockPilot MVP는 매일 08:00~15:30 사이 다음 단계가 자동 순차 실행되어야 한다:

1. 한투 접근 토큰 발급 (24시간 유효, 매일 재발급 필요)
2. DART 공시 + 한투 시세 수집
3. Claude API 분석 호출
4. 텔레그램 브리핑 발송
5. 사용자 승인 webhook 수신
6. 안전장치 검증 + 한투 주문 전송
7. 체결 폴링
8. 5분 간격 포트폴리오 재평가

1인 개발자가 모든 단계를 Python·Node 코드로 직접 작성·유지보수하기엔 부담이 크다. 오케스트레이션 도구로 각 단계를 노드·트리거 형태로 시각화·관리할 필요가 있다.

또한 이벤트 수집(Supabase INSERT) 훅을 각 단계에 심어야 하며, 실패 시 재시도·알림·로깅이 자동화되어야 한다.

---

## 검토한 대안

| 대안 | 장점 | 단점 |
|---|---|---|
| **n8n** | 셀프호스팅 가능, Fair-code, 노드 600+, HTTP Request로 모든 API 연동, 커뮤니티 활발 | 셀프호스팅 시 서버 관리 부담, Pro 유료 |
| Make.com | UI 직관적, SaaS | 월 실행 횟수 과금 → 월 6,000회 예상 시 비용 폭증, 한국 API(한투·DART) 전용 노드 없음 |
| Dify | LLM 워크플로우 특화 | 금융 API·텔레그램 연동 빈약, 생태계 작음 |
| Zapier | 가장 대중적 | 복잡한 조건부 분기·반복 루프 한계, 비용 |
| 직접 코딩 (Python + cron) | 완전 제어 | 1인 유지보수 부담, 시각화 X, 이벤트 훅 수동 구현 |

## 결정

**n8n (Pro 플랜 또는 셀프호스팅) 채택**

### 근거

1. **API 연동의 단순성**
   n8n의 HTTP Request 노드 하나로 모든 외부 API 연동 가능. 한투·DART·텔레그램·Supabase 모두 동일한 패턴으로 처리. Make.com 등은 서비스별 전용 노드에 의존해 한국 서비스 연동이 제한적.

2. **토큰 자동 재발급 처리 용이**
   한투 접근 토큰은 24시간만 유효해 매일 아침 재발급이 필수. n8n의 스케줄 트리거 + HTTP Request 조합으로 토큰 발급·저장·만료 감지·재발급 자동화가 단순함.

3. **이벤트 수집 통합 용이**
   각 워크플로우 노드 뒤에 Supabase INSERT 노드 1개를 추가하는 것만으로 이벤트 훅 심기 가능 (PRD 9조 이벤트 설계와 직결). 직접 코딩이면 매 함수마다 수동 DB 호출 필요.

4. **비용 예측성**
   월 실행 약 6,000회 예상. Make.com은 실행당 과금이라 비용 폭증. n8n Pro는 정액제, 셀프호스팅은 0원.

5. **포트폴리오 가치**
   n8n은 채용 시장에서 키워드 빈도 높음 ("AI Ops"·"워크플로우 자동화" 채용공고에 반복 등장).

## 결과·영향

**긍정**
- 복잡한 코드 없이 파이프라인 시각화·관리
- 토큰 재발급·이벤트 수집·재시도·알림 자동화 확보
- 포트폴리오 기술 스택 차별화

**부정**
- n8n 자체 학습 곡선 (초기 1~2일)
- 셀프호스팅 선택 시 서버 관리 필요
- 복잡한 분기 로직은 코드보다 가독성 떨어질 수 있음

**후속 작업**
- Day 2에 n8n 인프라 세팅 (Pro vs 셀프호스팅 선택은 Day 2 시작 시 판단)
- 모든 워크플로우에 Supabase INSERT 노드 심기 (Day 6)

## 재평가 조건

- 월 실행 횟수가 20,000회 초과 시 → 셀프호스팅 전환 비용 재검토
- n8n 이탈·Fair-code 정책 변경 시 → 대안 재평가
