# ADR-002: LLM으로 Claude Sonnet 4.6 선택

**날짜:** 2026-04-16
**상태:** Accepted
**관련:** PRD v1.5 섹션 5, 7 (AI 설계)

---

## 컨텍스트

StockPilot의 핵심 엔진은 LLM이다. 다음 요구사항을 충족해야 한다:

1. **시니어 애널리스트 수준 정성 분석** — 재무제표·공시 해석, 리스크 도출, 촉매 식별
2. **긴 문서 입력 처리** — DART 사업보고서·감사보고서 PDF 본문 직접 주입 (종목당 80K 토큰 수준)
3. **구조화된 출력 강제** — JSON 스키마로 환각 방지, 파싱 안정성
4. **비용 제어** — 월 $50 하드 리밋 내 운영 (종목 12개 × 매일 분석)
5. **프롬프트 캐싱** — 동일 문서 반복 참조 시 비용 절감
6. **한국어 번역** — 초보자용 3문장 요약

MVP는 AI 품질이 곧 제품 품질이므로 모델 선택이 결정적이다.

|---|---|---|---|
| **Claude Sonnet 4.6** | tool_use JSON 강제, 200K 컨텍스트, PDF 네이티브, prompt caching 90% 할인, extended thinking, 한국어 우수 | 없음 (현재 스펙상) | $3/1M |
| Claude Opus 4.6 | 최상 성능 | 비용 5배 ($15/1M), MVP엔 과사양, 월 $50 한도 초과 위험 | $15/1M |

## 결정

**Claude Sonnet 4.6 채택**

### 근거

1. **tool_use JSON 스키마 강제**
   PRD 7조의 핵심 원칙은 "환각 차단". Claude의 tool_use는 도구 스키마로 출력을 원천 강제해 마크다운 래핑·주석 삽입을 방지. GPT json_mode·Gemini는 이 수준의 엄격성 부족.

2. **PDF 네이티브 처리**
   DART 사업보고서·감사보고서를 PDF 본문 그대로 첨부해 분석 가능. 벡터DB·RAG·PDF 파싱 파이프라인 불필요 → MVP 규모에서 결정적 단순화 (PRD "직접 주입" 패턴 핵심).

3. **200K 컨텍스트 + prompt caching**
   종목당 문서 번들(사업보고서+감사보고서+수시공시) 80K 토큰을 한 번에 입력. 매일 반복되는 사업·감사보고서는 prompt caching으로 **90% 할인** 적용 → 월 비용 $13~18로 억제.

4. **비용과 성능의 균형**
   월 21M 토큰 예상 시:
   - Sonnet 4.6: ~$13~18 (캐싱 적용)
   - Opus 4.6: ~$65 (한도 초과)
   - Haiku 4.5: ~$4 (저렴하나 분석 품질 부족)
   Sonnet이 유일하게 품질·비용 양립.

5. **한국어 뉘앙스**
   "초보자도 이해할 3문장" 요약 품질이 다른 모델 대비 우수. 한국 금융 용어·공시 표현 처리 자연스러움.

6. **extended thinking**
   복잡한 재무 해석 시 내부 추론 단계를 거친 후 결론 도출. 단순 응답보다 품질 차원이 다름 (시니어 페르소나 유효성 핵심).

## 결과·영향

**긍정**
- AI 품질 확보 (시니어 애널리스트 페르소나 실효성)
- 월 비용 $50 한도 내 여유 확보
- PDF 직접 주입으로 RAG·벡터DB 생략 → 구현 단순화
- JSON 출력 안정성으로 파싱 실패 최소화

**부정**
- 단일 모델 의존 (Anthropic 장애 시 대안 없음)
- Anthropic 가격 정책 변경 리스크
- Opus 대비 최상위 종목 분석 깊이 아쉬울 수 있음

**후속 작업**
- Day 5에 시니어 페르소나 프롬프트 + tool_use 스키마 구현
- Day 8~9 A/B 실험으로 V1 vs V2 품질 검증
- 비용 모니터링: Anthropic 콘솔 월 $50 하드 리밋 설정

---

## 재평가 조건

- 월 비용이 $40 초과 → 캐싱 효율 재검토, Haiku 혼용 검토
- Sonnet 분석 품질이 A/B 실험에서 baseline 대비 개선 미흡 → Opus 부분 도입 검토 (핵심 종목 심층 분석 한정)
- Opus 가격 인하 또는 Haiku 성능 향상 → 재평가
- Phase 3 (실시간 뉴스 통합) 시 응답 속도 이슈 발생 시 → Haiku 분기 처리 검토

## 참고: Phase 2~3 하이브리드 검토

Phase 2 이후 데이터 축적되면:
- **일반 분석:** Sonnet 4.6 (기본)
- **심층 분석:** Opus 4.6 (주 1~2회, 핵심 종목만)
- **단순 요약:** Haiku 4.5 (알림 메시지 등)

MVP는 Sonnet 단일로 시작해 운영 데이터 기반 하이브리드 판단.
