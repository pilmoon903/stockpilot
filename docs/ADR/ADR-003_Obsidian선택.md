# ADR-003: 내부 문서 도구로 Obsidian 단일 채택

**날짜:** 2026-04-16
**상태:** Accepted
**관련:** PRD v1.5 섹션 5, 17 (프로젝트 관리)

---

## 컨텍스트

StockPilot은 1인 프로젝트지만 다음 문서 자산을 지속적으로 생성·관리해야 한다:

- PRD 버전 이력 (v1.0 → v1.5, 향후도 계속)
- ADR (의사결정 기록)
- Daily Journal (매일 작업 일기)
- Experiment Log (A/B 실험 기록)
- Hypothesis Table (가설·검증)
- Research Memo (baseline·2차 자료)
- Retro (주간 복기)
- 관심 종목 리스트
- 프롬프트 버전 관리

PM 취업 준비라는 Layer 2 목적상 **"문서 자산 = 포트폴리오"**이므로 도구 선택이 중요하다. 단, MVP 실행 속도를 해치지 않아야 하며, 도구 간 컨텍스트 전환 비용을 최소화해야 한다.

---

## 검토한 대안

| 대안 | 장점 | 단점 |
|---|---|---|
| **Obsidian 단일** | 로컬 Markdown, 빠른 속도, 백링크·그래프, 구글 드라이브 동기화로 멀티 디바이스, 오프라인, 0원 | 협업 약함, 공유 URL 없음 |
| Notion 단일 | 공유 URL·임베드 쉬움, DB·템플릿 강력, 팀 협업 | 오프라인 약함, 속도 느림, 데이터 락인 |
| Linear + Notion + Slack | 업계 표준 SaaS 조합 | 1인에겐 과잉, 컨텍스트 전환 3회, 월 비용 $20~30 |
| Obsidian + Notion (Hybrid) | 내부는 Obsidian, 공개는 Notion | 이중 관리 부담, 동기화 수동 |
| Google Docs | 접근성 최상 | PM 문서 구조 관리 빈약, 버전 관리 약함 |

---

## 결정

**Obsidian 단일 도구 채택. 단, 공개 채널용으로 Notion 별도 최소 사용.**

### 근거

1. **1인 MVP의 컨텍스트 전환 비용 최소화**
   Linear(태스크) + Notion(문서) + Slack(커뮤니케이션) 조합은 팀 협업엔 좋지만 1인에겐 3개 탭을 오가는 부담만 늘림. Obsidian 하나로 태스크·문서·로그·복기 모두 처리.

2. **로컬 Markdown의 실행 속도**
   Obsidian은 로컬 .md 파일. 열기 즉시 편집 가능, 검색 0.1초. Notion은 클라우드 의존 → 로딩·동기화 대기. 하루 수십 번 기록하는 MVP에선 속도 차이 누적.

3. **백링크·그래프로 자연스러운 의사결정 추적**
   [[ADR-001]], [[PRD_v1_5#7조]] 같은 백링크로 문서 간 연결. "왜 이 결정을 했는지" 클릭 한 번에 추적. ADR·PRD·Experiment Log 간 연결 표현에 최적.

4. **버전 관리 친화**
   Markdown이라 Git으로 그대로 버전 관리. PRD v1.0~v1.5가 파일로 남고, GitHub에도 그대로 푸시 가능 (PM 포트폴리오 자산).

5. **비용 0원**
   Notion Team: $10/월, Linear: $8/월, Slack Pro: $7/월. Obsidian은 무료(개인 용도). 구글 드라이브 동기화도 무료 15GB 안에서 해결.

6. **공개 분리 원칙**
   내부 문서는 Obsidian(비공개·속도), 외부 공개는 Notion(URL 공유·포트폴리오). 역할 분리로 각 도구의 강점만 활용.

## 결과·영향

**긍정**
- 단일 도구에서 모든 내부 작업 완결
- 로컬 속도 + 구글 드라이브 동기화로 멀티 디바이스 대응
- Markdown이라 GitHub·Notion으로 쉽게 이식
- 월 $25 이상 비용 절감

**부정**
- 협업 기능 거의 없음 (1인이라 문제 없음)
- 공유 URL을 위해 Notion 별도 운영 필요 (이중 관리 일부 발생)
- 모바일에서 편집 불편 (주로 데스크탑·노트북 작업)

**후속 작업**
- Day 1에 Notion 공개 페이지 스켈레톤 별도 작성 (Obsidian 내부 내용 중 공개할 것만 미러링)
- Obsidian → Notion 수동 미러링 주 1회 (주간 Retro 시)
- Obsidian 볼트 자체는 비공개 유지 (구글 드라이브 개인 저장소)

## 재평가 조건

- 프로젝트에 2인 이상 참여 → Notion 협업 기능 필요 시 재평가
- Obsidian 라이선스 정책 변경 → 대안 검토
- 공개 요구가 늘어나 Obsidian → Notion 미러링 부담이 과대해질 때 → Notion 중심 전환 검토
