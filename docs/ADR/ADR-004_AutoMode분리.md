# ADR-004: 자동매매(Auto 모드)를 Phase 2로 분리

**날짜:** 2026-04-16
**상태:** Accepted
**관련:** PRD v1.5 섹션 3, 16 (솔루션 철학, 확장 로드맵)

---

## 컨텍스트

StockPilot의 기술 아키텍처는 이미 완전 자동매매의 95%를 갖추고 있다. 빠진 것은 "인간 승인 단계 제거 + 자동 트리거 조건"뿐이다. 기술적으로는 Phase 1 MVP에서도 구현 가능하다.

그러나 MVP 단계에서 자동매매를 도입할지 여부는 기술 문제가 아닌 **제품 정체성 문제**다. 다음 5가지가 충돌한다:

1. **Archetype C 정의** — "AI 판단 참고하되 승인은 내가" (PRD 2조)
2. **솔루션 철학** — "AI가 분석·번역, 사람이 결정, 시스템이 체결" (PRD 3조)
3. **NSM 측정 방식** — Decision Quality Lift는 인간 결정이 있어야 측정 가능 (PRD 10조)
4. **규제 리스크** — 본인 1인 사용은 합법이나 확장 경로 명확히 구분 필요 (PRD 15조)
5. **포트폴리오 서사** — "HITL로 통제감 해결"이 면접 핵심 메시지

---

## 검토한 대안

| 대안 | 장점 | 단점 |
|---|---|---|
| **Phase 2 분리 (토글 옵션)** | MVP 철학 유지, 검증 후 확장, ADR 서사 강력 | Phase 2까지 대기 |
| MVP 포함 (HITL + Auto 혼합) | 실사용 편의, 기술 완성도 | Archetype C 정의 붕괴, KPI 측정 불가, PM 서사 혼란 |
| 자동매매 배제 (영구) | 일관성 | 실사용 가치 성장 한계 |

---

## 결정

**Auto 모드는 Phase 2에 옵션 토글로 도입. MVP(Phase 1)는 HITL 유지.**

### 근거

1. **Archetype 정의 일관성**
   MVP는 통제지향 Archetype C만 정조준. 자동매매 도입 시 C 정의 자체가 무너짐. "통제를 원하면서 자동화도 허용"은 자기모순.

2. **NSM 측정 가능성**
   Decision Quality Lift는 "brif 있는 내 결정 vs brif 없는 baseline 결정" 비교. 인간이 빠지면 Lift 측정 구조 자체가 성립 불가. MVP의 핵심 측정 프레임 보호.

3. **AI 신뢰 데이터 축적 필요**
   10영업일 HITL 운영 후 confidence 구간별 실제 수익률 데이터가 쌓여야 "어떤 조건에서 자동 허용할지" 판단 가능. 데이터 없이 자동매매 도입은 근거 없는 모험.

4. **규제 리스크 구분 명확화**
   본인 1인 HITL과 본인 1인 Auto는 모두 합법이나, Auto 모드를 타인에 서비스로 제공하면 즉시 투자일임업 라이선스 필요. HITL 기반 MVP로 먼저 스코프를 명확히 한 뒤 Auto 토글 추가가 규제 리스크 관리에 유리.

5. **포트폴리오 서사 강화**
   "기술적으로 가능했지만 의도적으로 분리한 판단" 자체가 PM 역량 증거. 면접에서 "왜 자동화 안 했나요?" 질문에 철학·측정·규제 3축 근거로 답변 가능. 반대로 "다 자동화했습니다"는 PM 판단력 약화 시그널.

## Phase 2 도입 시 설계 방향 (예비)

### Auto 모드 토글 조건 (예상)
```
기본 모드 (HITL): 모든 종목 수동 승인
Auto 모드 선택 시:
  조건: confidence ≥ 0.85 AND risk_count ≤ 2 AND 운영일 30일+
  대상: 선택한 종목만 (전체 적용 X)
  긴급 중지: /pause_auto 명령어
```

### 신뢰 캘리브레이션 곡선 (Trust Curve)
```
10영업일 HITL 데이터 → confidence 구간별 실제 수익률 산출
→ Auto 허용 임계치 = 수익률 안정적 구간의 하한
```

### 진입 조건 (Phase 2 시작 가능 기준)
- MVP 10영업일 + 모의 4주 관측 완료
- Decision Quality Lift 안정화
- Guardrail Breach Rate 0 유지
- False Positive Cost 평균 regret_score < 2.5

## 결과·영향

**긍정**
- MVP 정체성 보호 (Archetype C, NSM, 규제 스코프 일관)
- Auto 도입 판단의 근거 데이터 확보 (10영업일+)
- PM 서사 강화 ("분리 판단 자체가 역량 증거")

**부정**
- 실사용 편의성 일부 희생 (매일 승인 부담)
- Phase 2까지 기능 지연 (최소 4주)
- Auto 도입 준비 추가 작업 (Phase 2에 별도 안전장치·토글 설계)

**후속 작업**
- MVP 10영업일 + 모의 4주 완료 후 Trust Curve 산출
- Phase 2 진입 시점에 Auto 모드 토글 ADR 별도 작성 (ADR-00X)

---

## 재평가 조건

- MVP 10영업일 결과 Decision Quality Lift < 0 → Auto 도입보다 AI 분석 개선 우선
- MVP에서 HITL 승인 누락률 > 50% → Auto 도입 시점 앞당기기 검토
- 규제 환경 변화 (투자일임업 1인 허용 등) → Auto 확장 범위 재평가
- Claude 성능 급격히 향상 → 신뢰 임계치 완화 검토

## 면접 답변 프레임 (참고)

**Q. 왜 자동매매로 안 갔나요?**

A: "세 가지 이유로 Phase 2로 의도적으로 분리했습니다. 첫째, 타깃 Archetype이 통제지향 사용자라 자동매매와 철학적으로 충돌합니다. 둘째, NSM인 Decision Quality Lift는 인간 결정이 있어야 측정 가능해 Auto 모드 도입 시 측정 구조가 무너집니다. 셋째, 규제 스코프 관리 측면에서 HITL 스코프를 먼저 확정한 뒤 Auto는 옵션으로 추가하는 것이 안전합니다. 기술적으로는 가능했지만, **이 분리 판단 자체가 제품 의사결정의 핵심**이라고 생각했습니다."
