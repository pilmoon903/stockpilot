## 상위 프레임 🆕 (v1.5.1 정합)

### 이 가설들이 검증하는 것
**주가 예측 시스템의 정확도가 아닌, 의사결정 품질 개선 시스템의 유효성.**

아래 3개 가설은 모두 **"AI가 주가를 맞추는가"가 아니라 "AI가 본인의 의사결정 프로세스를 개선하는가"**에 초점.

| 상위 질문 | 검증 가설 |
|---|---|
| 체크리스트 자동화가 효율성을 높이는가? | H1 (분석 시간) |
| HITL 구조가 실행을 개선하는가? | H2 (전환율) |
| 2차 관점(confidence)이 판단을 개선하는가? | H3 (후회 감소) |

### 검증하지 않는 것 (의도적)
- ❌ "AI 추천이 실제로 수익을 내는가"
- ❌ "이 시스템 수익률 > 코스피 벤치마크인가"
- ❌ "confidence 높은 종목이 실제 상승하는가"

**이유:** 주가는 예측 불가하므로 이런 가설은 소표본에선 운에 좌우됨. **통제 가능한 '본인의 의사결정 프로세스'**만 측정 대상.

## H1: 체크리스트 자동화로 분석 시간 단축

### 가설
**"AI 브리핑 사용 시, 본인이 종목을 분석·결정하는 데 드는 시간이 baseline 대비 70% 단축된다."**

### 상위 프레임 연결
**"체크리스트 자동화" 가치의 정량 검증.** AI가 수행하는 체크리스트(재무·공시 행간·운전자본·밸류에이션)가 본인 수동 작업을 대체하는 정도를 측정.

### 근거
- PRD 1조 정량 문제 진술: 개인 투자자 분석 시간 45~60분/일
- Claude 시니어 페르소나가 수동 체크리스트를 대체 → 본인은 "확인·판단"만
- 2차 자료에서 "해석 단계"가 가장 오래 걸린다는 일관된 관찰

### Baseline 측정
- **방법:** Day 0~1 동안 본인이 관심종목 3개를 직접 분석했을 때 걸린 시간 측정
- **기록 방식:** Obsidian Daily Journal의 "Baseline 측정" 섹션
- **Baseline 값:** TBD → 측정 후 기록 (예상: 30~60분/종목)

### 측정 방법 (Day 8~9)
- `decision_made` 이벤트 중 `decision_latency_sec` 집계
- "브리핑 열람 → 결정"까지 걸린 시간 vs baseline 비교

### 성공 기준
- **완전 성공:** baseline의 30% 이하 (-70%)
- **부분 성공:** baseline의 50% 이하 (-50%)
- **실패:** baseline의 70% 이상

### 실패 시 행동
- 프롬프트 단순화 (불필요한 정보 제거)
- 브리핑 요약 길이 축소 (3문장 → 2문장)
- "펼쳐보기" 기본 접힘으로 정보량 제어

### 위험
- **시간 단축 ≠ 결정 품질.** H3와 교차 검증 필수
- baseline이 과잉 측정될 가능성 (인프라 세팅 시간 포함 혼동)

## H2: HITL 구조가 실행 전환율 향상

### 가설
**"텔레그램 브리핑 + [매수] 버튼 구조는 외부 결정(HTS 따로 열기)보다 실행 전환율을 높인다."**

### 상위 프레임 연결
**"HITL 존재 이유" 검증.** AI가 체크리스트를 대체하고 최종 판단만 남은 상황에서, 판단-실행 갭을 최소화하는 구조가 실제 작동하는지 측정.

### 근거
- 외부 앱 열기·로그인·종목 검색·주문 입력 → 각 단계마다 이탈
- 버튼 1회 클릭은 인지 부담 최소
- 단타 트레이더의 조건검색식 → 단축키 매수 구조와 유사 원리

### Baseline 측정
- **방법:** 본인 과거 1~3개월 매수 기록 회상 — "결정부터 실제 매수까지 평균 몇 시간/며칠?"
- **Baseline 값:** TBD (예상: 평균 1~3일)
- ⚠️ 회상 기반이라 N=1·정확도 낮음 명시

### 측정 방법
- `decision_made` → `order_executed` 전환률 (%)
- 전환 소요 시간 평균
- 상태별 분류: 체결 / 미체결 / 거절로 남음

### 성공 기준
- **완전 성공:** 결정 후 5분 내 체결률 90% 이상
- **부분 성공:** 30분 내 체결률 80% 이상
- **실패:** 결정 후 체결률 50% 미만 or 5분 평균 지연

### 실패 시 행동
- UX 단순화 (버튼 레이블 명확화)
- 안전장치 통과 속도 개선
- 체결 폴링 주기 단축

### 위험
- **실행률 높음 ≠ 좋은 결정.** 성급한 실행 유도는 오히려 해로움 → H3 교차 검증 필수
- HITL의 목적은 '속도'가 아닌 '통제감'. 속도 지표만 강조 시 본말전도

## H3: 2차 관점(confidence)이 거절 품질 향상 (A/B 실험)

### 가설
**"브리핑에 confidence 수치가 표시되면, 부적절한 매수 권고를 더 잘 거절해 사후 후회가 감소한다."**

### 상위 프레임 연결
**"편향 없는 2차 관점" 가치의 정량 검증.** AI가 제공하는 confidence는 본인 편향에 대한 외부 신호. 이 신호가 실제로 판단을 교정하는지 측정.

### 근거
- Trust Calibration 이론: 신뢰도 가시화 → 판단 신중도 증가
- 낮은 confidence 항목은 "AI도 확신 없음" 신호로 작용
- 블랙스완은 AI도 모르지만, **"AI가 애매하다고 하는 것"은 경고 신호로 기능 가능**

### Baseline 측정
- Day 8 오전 (또는 첫 3영업일) A안(confidence 미표시)으로 운영
- A안 그룹의 평균 regret_score (T+3 자가 평가)

### A/B 설계
- **A (Control):** 브리핑에 confidence 미표시, 결정 버튼만
- **B (Treatment):** confidence 수치 표시 (0.0~1.0) + 라벨 (긴급/검토/참고)
- **분배:** 종목 홀짝 분할 (홀수 코드 → A, 짝수 → B)

### 측정 방법
- 각 그룹의 `post_decision_regret` 평균 (1~5점)
- 낮은 confidence(<0.7) 항목의 거절률 A vs B 비교
- 거절한 항목의 실제 사후 수익률 비교 (참고용, 주 판정 아님)

### 성공 기준
- **완전 성공:** B그룹 regret_score 평균 -1.0점 이상
- **부분 성공:** -0.5점 이상
- **실패:** 차이 없음 (< -0.2) 또는 역효과

### 실패 시 행동
- confidence 표기 방식 재설계 (수치 → 3단계 라벨만)
- confidence 아예 제거
- 다른 신호 시도 (리스크 개수 강조 등)

### 위험
- **소표본 한계.** 10영업일 × 12종목 ÷ 2그룹 ≈ 60건 이하. 통계적 유의성 X
- "PM 서사용 실험"임을 면접에서 솔직히 인정 필요
- A/B의 핵심은 수치가 아닌 **"가설→측정→학습" 사이클 경험**
